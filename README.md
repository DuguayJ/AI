# Universal Document Builder

Generate professional, branded Word (.docx) and Excel (.xlsx) documents from
structured JSON data — with consistent colour palette, headers, footers, and
block-based content composition.

---

## Requirements

```
Node.js  >= 16        (for Word generation)
Python   >= 3.9       (for CLI + Excel generation)
npm packages:  docx   (npm install -g docx)
pip packages:  openpyxl
```

Install once:
```bash
npm install -g docx
pip install openpyxl --break-system-packages
```

---

## Files

| File | Purpose |
|---|---|
| `document_builder.py` | Main entry point — interactive wizard, JSON mode, Excel builder |
| `build_word.js` | Node.js Word (.docx) generator — called internally by the Python script |
| `example_input.json` | Generated by `--example` flag — ready-to-edit template |

---

## Quick Start

### Interactive wizard
```bash
python3 document_builder.py
```
Walks you through metadata, sections, and blocks step by step.

### From a JSON file
```bash
python3 document_builder.py --example          # writes example_input.json
python3 document_builder.py --json example_input.json
```

### Direct Node.js (Word only)
```bash
node build_word.js '{"reportTitle":"My Doc", "sections":[...]}' output/my_doc.docx
```

## JSON Schema

```jsonc
{
  // ── Metadata ──────────────────────────────────────────────────────────
  "reportTitle":    "Document Title",           // required
  "documentType":   "IT Security",              // shown as label on cover
  "subtitle":       "Scope or description",
  "organization":   "Acme Corp",
  "environment":    "Production — North DC",
  "date":           "2025-04-30",
  "classification": "Confidential — Internal Use Only",
  "preparedBy":     "IT Security Team",
  "version":        "1.0",
  "status":         "Draft | In Progress | Pending Review | Completed",
  "includeToc":     false,                      // Word only

  // ── Sections ──────────────────────────────────────────────────────────
  "sections": [
    {
      "title": "Section Title",
      "blocks": [ /* see Block Types below */ ]
    }
  ]
}
```

## Block Types

All blocks sit inside a section's `"blocks"` array.

### `text`
Plain paragraph.
```json
{ "type": "text", "content": "Paragraph text here." }
```

### `heading2`
Sub-heading within a section.
```json
{ "type": "heading2", "content": "2.1 Sub-section title" }
```

### `bullets`
Unordered list.
```json
{ "type": "bullets", "items": ["First item", "Second item"] }
```

### `numbered`
Ordered list.
```json
{ "type": "numbered", "items": ["Step one", "Step two"] }
```

### `callout`
Highlighted notice box. Styles: `info` · `warning` · `error` · `success`
```json
{ "type": "callout", "style": "warning", "content": "Action required by May 31." }
```

### `table`
Data table with styled header row.
```json
{
  "type": "table",
  "columns": ["Category", "NDH", "HPH"],
  "rows": [
    ["Patch Compliance", "71%", "65%"],
    ["EDR Coverage",     "92%", "88%"]
  ]
}
```

### `key_value`
Two-column label / value grid.
```json
{
  "type": "key_value",
  "items": [
    ["Scope",       "All Windows and Linux endpoints"],
    ["Lead",        "J. Tremblay, CISSP"],
    ["Findings",    "12 (3 Critical · 5 High · 4 Medium)"]
  ]
}
```

## Status Colours

| Status | Colour |
|---|---|
| `Completed` | Teal / Green |
| `In Progress` | Amber / Yellow |
| `Pending Review` | Blue |
| `Draft` | Gray |

## Use Cases

| Document Type | Suggested Blocks |
|---|---|
| Security Assessment | key_value metadata · table findings · callout warnings · numbered next steps |
| Asset Registry | table inventory · callout EOL notice · key_value summary |
| SOP / Procedure | heading2 phases · bullets steps · callout critical notes · numbered actions |
| Project Status | key_value KPIs · table milestones · callout blockers |
| Audit Report | table controls · key_value scope · bullets findings · numbered recommendations |
